Getting Started

To begin your journey with Vertigo, you'll need the Vertigo executable. Vertigo programs are written in plain text files, typically saved with a .vtgo extension.

1. Obtain Vertigo and dizzypack

Vertigo is distributed as a standalone executable (vertigo.exe on Windows), compiled using PyInstaller. For managing Vertigo and its libraries, we provide dizzypack, a simple command-line package manager, also distributed as an executable (dizzypack.exe).

Visit the Downloads page to get the latest versions of vertigo.exe and dizzypack.exe.

2. Set up your Environment (Recommended)

For ease of use, it's highly recommended to add the directory containing vertigo.exe and dizzypack.exe to your system's PATH environment variable. This allows you to run both commands from any directory in your terminal or command prompt.

(Instructions for adding to PATH vary by operating system. A quick web search for "how to add to PATH [your OS]" will provide detailed steps.)

3. Install Libraries with dizzypack

Once `dizzypack.exe` is set up, you can use it to install external Vertigo libraries or Python modules that Vertigo can import.

Example: Installing a Library (e.g., `amath`)

Open your terminal/command prompt and run:
dizzypack amath

This command will download the `amath.py` module into the `libs` directory located alongside your `vertigo.exe` and `dizzypack.exe`. You can then use the `IMPORT "amath"` instruction in your Vertigo code to utilize it.

Updating Vertigo:

To update your Vertigo interpreter to the latest version, use:
dizzypack update

This command will download the newest `vertigo.exe` and replace your current one, creating a backup of the old version.

4. Your First Vertigo Program: Hello World

Let's create a simple "Hello, World!" program to demonstrate the basic syntax and execution flow in Vertigo. 

; Initialize a new stack named 'main_stack' for our primary operations
NEW main_stack

; Set 'main_stack' as the currently active stack
main_stack

; Push the classic greeting string onto the top of the stack
PUSH "Hello, World!"

; Transfer the value from the top of the stack into the Output Data Accumulator (ODA) register
POP ODA

Code Breakdown:

  • NEW main_stack: Creates a new, empty stack named `main_stack`. Vertigo heavily relies on stacks for data manipulation.
  • main_stack: This line simply selects `main_stack` as the currently active stack. Subsequent stack operations will apply to this stack.
  • PUSH "Hello, World!": Places the string literal "Hello, World!" onto the top of `main_stack`.
  • POP ODA: Takes the value from the top of the current stack (`"Hello, World!"`) and moves it into the special `ODA` (Output Data Accumulator) register. When a value is placed into `ODA` (and the `intpr` setting is `FALSE`, which is the default), Vertigo will automatically print its content to the console at the end of the current instruction's execution.

Regarding `INT` instructions:

While `INT` instructions exist in Vertigo, they are for specific, low-level interactions or interrupt handlers. They are generally not needed for basic program output or normal program termination.

  • INT 0x0: This interrupt is designed for situations where you need to explicitly exit the program prematurely, for instance, due to an error condition or a specific control flow requirement. For a typical program's natural end, it's not required.
  • INT 0x1: This interrupt's primary function is to print the contents of the `ODA` register, but only when the interpreter's internal `intpr` setting is explicitly set to `TRUE`. This provides a controlled way to print if default `ODA` printing behavior is altered or when `intpr` mode is desired. In standard operation (with `intpr` as `FALSE`), simply placing a value into `ODA` will print it.

5. Running Your Program

Save the code snippet above as `hello.vtgo` in the same directory as your `vertigo.exe` executable. Then, open your terminal or command prompt, navigate to that directory, and execute the program using the following command:

vertigo hello.vtgo

You should see "Hello, World!" printed to your console!

You're now ready to explore more of Vertigo's powerful instructions and build more complex programs. Check out the Instruction Reference for a full list of commands.
